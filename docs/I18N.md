# 多语言(i18n)— 放语言文件即生效

设计原则:**贡献一门语言 = 放文件,不改代码**。三端各有一个"投放点"。

## 1. 电脑端播报(agent)

**投放点:`agent/lang/<BCP47>.json`**(如 `ja-JP.json`、`fr-FR.json`)

复制 `agent/lang/zh-CN.json` 翻译即可。结构:

```json
{
  "meta": {
    "name": "日本語",
    "speechLocale": "ja-JP",
    "edgeVoice": "ja-JP-NanamiNeural"
  },
  "tools":   { "Bash": "コマンド実行" },
  "gui":     { "screenshot": "スクリーンショット" },
  "phrases": { "received": "了解、{prompt}" }
}
```

规则:
- **缺键自动回落 zh-CN** —— 翻一半也能跑,但 `npm test` 的键齐全测试会列出缺键(交 PR 前补齐)
- `{prompt}`、`{label}` 等占位符**原样保留**,只翻周围文字;占位符位置可按目标语言语序移动
- meta 三项:`name` 显示名、`speechLocale` 配套语音识别 locale、`edgeVoice` 配套云端音色建议

**启用**:`~/.earpiece/settings.json` 加 `"lang": "ja-JP"` 后重启 daemon;或环境变量 `EARPIECE_LANG=ja-JP`。

## 2. Android App

**投放点:`android/app/src/main/res/values-<语言>/strings.xml`**(如 `values-ja/`)

复制 `values/strings.xml`(中文基准)整翻,Android 原生机制随系统语言自动切换。两个特殊键一起改:
- `tts_locale` — 系统 TTS 朗读语言(如 `ja-JP`)
- `edge_default_voice` — 微软云端默认音色(如 `ja-JP-NanamiNeural`)

## 3. iOS App

- **语音输入**:设置 → 「语音识别语言」填 BCP47 码(如 `en-US`),留空=跟系统语言;每次开录即生效(`SFSpeechRecognizer` locale)
- 界面文案 `.lproj` 本地化:规划中(届时投放点 `spike/ios/<语言>.lproj/Localizable.strings`)

## 4. 语音输入(上行)要点

- 项目名是路由键,说指令时项目名照原文念(英文项目名就说英文)
- 指令本身用什么语言由你的 CLI 模型决定 —— Claude/Codex/Hermes 本来就多语言
- Android v1 暂无麦克风输入(规划中);语言包 `meta.speechLocale` 是给未来识别器的约定位

## 键齐全校验

`npm test` 含硬测试:`agent/lang/` 下每个包必须覆盖 zh-CN 全部键,缺键报红并列出键名。交语言包 PR 前先跑。
